## 四、使用流程

### 步骤1：填写变量
参照 `05_要素录入表.md` 中的字段清单，填写目标基金的各项要素。

### 步骤2：确认条件标志
根据基金类型，确认以下布尔标志：
- `HAS_HK_CONNECT`：是否为港股通基金
- `HAS_AML`：是否含反洗钱条款（目前仅交通银行托管版本含）
- `HAS_CUSTODIAN_TRANSFER_SPECIAL`：是否含托管子公司变更程序（目前仅中信证券托管版本含）
- `HAS_STOCK_SUBSCRIPTION`：是否支持网下股票认购（深交所上市通常有）

### 步骤3：生成合同
用模板引擎读取 `01_通用模板.md`，按以下规则处理：
1. 将所有 `{变量名}` 替换为对应值
2. 根据条件标志决定 `{{IF ...}}...{{ENDIF}}` 块是否输出
3. 参照 `04_差异条款原文库.json` 填充差异条款的具体原文

### 步骤4：验证
对照 `03_差异条款匹配表.md` 中的关键条款，逐项检查生成结果。

## 六、注意事项

1. **固定条款保留原文**：模板中所有固定法律文本均来自合同原文，未作修改。
2. **南方基金管理人信息**：管理人信息（地址、注册资本等）已设为默认值，但设计为可替换变量。
3. **港股通基金利润定义**：港股通合同含"汇兑损益"，须使用 `{{IF HAS_HK_CONNECT}}` 条件块区分。
4. **反洗钱条款**：目前仅在电池主题（交通银行托管）版本中发现，其他合同无第27-29条，需谨慎核对。
5. **收益分配第2条**：dianchi合同无"每月"/"每季度"限定，模板中对应的变量为 `DISTRIBUTION_FREQ=NONE`。
6. **托管子公司程序**：仅红利质量（中信证券托管）含（四）特殊程序，约230字，已完整录入原文库。
