# 基金合同要素录入表（UI设计规范）

**用途：** 开发人员构建基金合同自动生成系统时参考的字段规范，定义界面字段顺序、输入类型、联动规则等。

---

## 第一组：基金基本信息

| # | 字段名（中文） | 变量名 | 输入类型 | 是否必填 | 默认值/选项 | 填写说明 |
|---|-------------|-------|---------|---------|-----------|---------|
| 1 | 基金全名 | FUND_NAME | 文本框（长）| ✓ | 无 | 格式：南方XXXX交易型开放式指数证券投资基金 |
| 2 | 基金简称 | FUND_SHORT_NAME | 文本框（中）| ✓ | 无 | 如：南方创业板中盘200ETF |
| 3 | 基金代码 | FUND_CODE | 文本框（短，6位数字）| 否 | 无 | 上市后确认，募集前可留空 |
| 4 | 标的指数名称 | INDEX_NAME | 文本框（中）| ✓ | 无 | 如：创业板中盘200指数 |
| 5 | 标的指数代码 | INDEX_CODE | 文本框（短）| ✓ | 无 | 如：399371（创业板）/ 000688（科创50） |
| 6 | 指数编制机构 | INDEX_COMPILER | 文本框（中）| 否 | 自动推断 | 如：深圳证券交易所 / 中证指数有限公司 |
| 7 | 合同签署日期 | CONTRACT_DATE | 月份选择器 | ✓ | 当月 | 格式：XXXX年X月 |

## 第二组：交易所与市场类型

**⚠️ 此组字段存在联动规则，会自动填充多个下游字段**

| # | 字段名（中文） | 变量名 | 输入类型 | 是否必填 | 选项 | 填写说明 |
|---|-------------|-------|---------|---------|-----|---------|
| 8 | 上市交易所 | EXCHANGE | 下拉框 | ✓ | 深圳证券交易所 (SZSE) / 上海证券交易所 (SSE) | **联动触发项** |
| 9 | 投资市场类型 | MARKET_TYPE | 下拉框 | ✓ | 普通A股 (A_SHARE) / 创业板 (CHUANGYE) / 科创板 (KECHUANG) / 港股通 (HK_CONNECT) | **联动触发项** |
| 10 | 工作日定义类型 | WORKING_DAY_TYPE | 下拉框 | ✓ | 仅深交所 / 仅上交所 / 沪深两所 | **通常自动推导，可手动调整** |
| 11 | 《业务规则》类型 | BUSINESS_RULES_TYPE | 下拉框 | ✓ | 深交所版 (SZSE) / 上交所版 (SSE) | **通常自动推导** |

### 联动规则说明

```
当 EXCHANGE = 深交所 (SZSE) 时：
  → WORKING_DAY_TYPE  = SZSE_ONLY（可调整）
  → BUSINESS_RULES_TYPE = SZSE（自动）
  → EXCHANGE_NAME_CN = "深圳证券交易所"（自动）

当 EXCHANGE = 上交所 (SSE) 时：
  → WORKING_DAY_TYPE  = SSE_ONLY（可调整）
  → BUSINESS_RULES_TYPE = SSE（自动）
  → EXCHANGE_NAME_CN = "上海证券交易所"（自动）

当 MARKET_TYPE = 港股通 (HK_CONNECT) 时：
  → HAS_HK_CONNECT    = true（自动）
  → TRACKING_ERROR_DAILY  = 0.35（默认，可调整）
  → TRACKING_ERROR_ANNUAL = 4（默认，可调整）
  → HAS_STOCK_SUBSCRIPTION = false（默认，通常不变）

当 MARKET_TYPE = 普通A股 / 创业板 / 科创板 时：
  → HAS_HK_CONNECT    = false（自动）
  → TRACKING_ERROR_DAILY  = 0.2（默认，可调整）
  → TRACKING_ERROR_ANNUAL = 2（默认，可调整）
  → NON_COMPONENT_SCOPE = 自动填充对应文本（可查看预览）

当 EXCHANGE = 深交所 且 MARKET_TYPE ≠ 港股通 时：
  → HAS_STOCK_SUBSCRIPTION = true（推荐默认，可调整）
```

## UI设计建议

### 分步表单推荐结构

1. **步骤一：基本信息**（字段1-7）
2. **步骤二：交易所与市场**（字段8-11）+ 联动预览
3. **步骤三：托管人选择**（字段12）→ 自动展开 13-21
4. **步骤四：费用与分配**（字段31-36）
5. **步骤五：特殊条款确认**（字段37-40）+ 警告提示
6. **步骤六：预览与生成**（显示合同摘要，标注差异条款位置）

### 差异条款实时预览建议

在编辑表单时，右侧面板应实时显示受影响条款的原文预览，特别是：
- 第二部分释义第39条（工作日）
- 第二部分释义第44条（业务规则）
- 第十四部分第三条（跟踪误差）
- 第十七部分第二条（费用划款）
- 第十八部分第三条第2款（收益分配频率）

### 验证规则

| 字段组合 | 验证规则 | 错误提示 |
|---------|---------|---------|
| EXCHANGE=SSE 且 WORKING_DAY_TYPE=SZSE_ONLY | 警告 | 上交所上市基金建议使用上交所或沪深两所工作日定义 |
| MARKET_TYPE=HK_CONNECT 且 TRACKING_ERROR_ANNUAL < 4 | 警告 | 港股通基金年跟踪误差通常设为4%，请确认 |
| MARKET_TYPE=HK_CONNECT 且 HAS_HK_CONNECT=false | 错误 | 港股通市场类型必须同时勾选港股通标志 |
| MGMT_FEE_RATE = 0 | 错误 | 管理费率不得为零 |
| DISTRIBUTION_FREQ 未选择 | 错误 | 必须选择收益分配评估频率 |

## 附：chuangye参数验证示例

> 使用以下参数填入后，生成的合同应与原始 `南方创业板中盘200交易型开放式指数证券投资基金基金合同.docx` 一致。

```
FUND_NAME              = 南方创业板中盘200交易型开放式指数证券投资基金
FUND_SHORT_NAME        = 南方创业板中盘200ETF
INDEX_NAME             = 创业板中盘200指数
INDEX_CODE             = 399371（请核对实际代码）
CONTRACT_DATE          = 2025年6月
EXCHANGE               = SZSE（深圳证券交易所）
MARKET_TYPE            = CHUANGYE（创业板）
WORKING_DAY_TYPE       = SZSE_ONLY
BUSINESS_RULES_TYPE    = SZSE
CUSTODIAN_NAME         = 招商银行股份有限公司
TRACKING_ERROR_DAILY   = 0.2
TRACKING_ERROR_ANNUAL  = 2
MGMT_FEE_RATE          = 0.15
CUSTODY_FEE_RATE       = 0.05
MGMT_FEE_PAYMENT_METHOD  = CONSULT
DISTRIBUTION_FREQ      = MONTHLY
HAS_HK_CONNECT         = false
HAS_AML                = false
HAS_CUSTODIAN_TRANSFER_SPECIAL = false
HAS_STOCK_SUBSCRIPTION = true
```

**验证关键条款：**
1. 释义39条 → "指深圳证券交易所的正常交易日" ✓
2. 释义44条 → 深交所版《业务规则》 ✓
3. 第四部分发售 → "3种方式"含网下股票认购 ✓
4. 第十四部分三 → "日均跟踪偏离度不超过0.2%，年跟踪误差不超过2%" ✓
5. 第十七部分一 → 11项（无港股通费用） ✓
6. 第十七部分二 → "协商一致的方式于次月前5个工作日内" ✓
7. 第十八部分三 → "可每月评估"、"可每月对" ✓
8. 第九部分 → 管理人义务无27-29条（无反洗钱条款） ✓
9. 第十一部分二 → 无（四）特殊程序 ✓
